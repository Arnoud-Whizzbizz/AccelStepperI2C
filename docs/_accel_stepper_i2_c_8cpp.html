◆ resetAccelStepperSlave()

void resetAccelStepperSlave ( uint8_t  address)

Tells the slave to reset. Call this before adding any steppers if master and slave resets are not synchronized by hardware. Else the slave survives the master's reset, will think the master wants another stepper, not a first one, and will run out of steppers, sooner or later.

All steppers are stopped by AccelStepper::stop() and disabled by AccelStepper::disableOutputs(), after that a hardware reset is triggered.

◆ setI2Cdelay()

uint16_t setI2Cdelay ( uint16_t  delay)

Define a delay that the master waits each time before sending a command to the slave and before requesting a reply from it. This is to make sure that the slave has finished its earlier task or has its answer to the master's previous command ready. Particularly for the ESP32 this is critical, as due to its implementation of I2C slave mode, it could theoretically send incomplete data if a request is sent too early.

Parameters
delayTime to wait in milliseconds.
Returns
Returns the previously set delay.
