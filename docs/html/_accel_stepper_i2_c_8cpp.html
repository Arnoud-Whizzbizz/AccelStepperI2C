void resetAccelStepperSlave ( uint8_t  address)

Tells the slave to reset. Call this before adding any steppers if master and slave resets are not synchronized by hardware. Else the slave survives the master's reset, will think the master wants another stepper, not a first one, and will run out of steppers, sooner or later.

All steppers are stopped by AccelStepper::stop() and disabled by
AccelStepper::disableOutputs(), after that a hardware reset is triggered.
